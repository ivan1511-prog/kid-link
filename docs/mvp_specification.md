
## MVP: Proximity Watch — Спецификация v1.0

### Цель
Двое часов T-Watch-S3 оповещают вибрацией при удалении друг от друга на ~2 метра.

---

### Архитектура

Поскольку вибрировать должны **обе** пары часов, каждое устройство работает одновременно как **Beacon + Scanner**.

```
┌─────────────────┐         BLE          ┌─────────────────┐
│   РОДИТЕЛЬ      │ ◄──────────────────► │   РЕБЁНОК       │
│                 │                      │                 │
│  [Beacon]       │   advertisements     │  [Beacon]       │
│  [Scanner]      │ ◄──────────────────► │  [Scanner]      │
│  [Vibro]        │                      │  [Vibro]        │
│  [Display]      │                      │  [Display]      │
└─────────────────┘                      └─────────────────┘
```

**Идентификация пары:** UUID в BLE advertisement (одинаковый на обоих часах, прошивается при сборке).

### Логика работы

| Состояние | RSSI | Расстояние | Действие |
|-----------|------|------------|----------|
| OK | > -65 dBm | < ~2 м | 🟢 на экране |
| FAR | -65 ... -80 dBm | ~2-5 м | 🔴 + вибрация |
| NO SIGNAL | нет данных 3 сек | — | ⚠️ + непрерывная вибрация |

**Вибрация:**
- При FAR: 300 мс вибро, повтор каждые 3 сек (пока далеко)
- При NO SIGNAL: 200 мс вибро, повтор каждые 1 сек

---

### Технические параметры

| Параметр | Значение |
|----------|----------|
| BLE Advertisement interval | 200 мс |
| Scan interval | 500 мс |
| RSSI averaging window | 5 измерений (скользящее среднее) |
| Порог FAR | -65 dBm (≈2 м, калибруется) |
| Порог NO SIGNAL | нет данных > 3 сек |
| UUID пары | фиксированный при прошивке |

---

### Преобразование RSSI → расстояние

Формула (приблизительная):
```
distance = 10 ^ ((TxPower - RSSI) / (10 * n))
```
где `TxPower` ≈ -59 dBm (на 1 м), `n` ≈ 2 (коэффициент среды)

**Отображаем как:** `~0.5 m`, `~1 m`, `~2 m`, `~3+ m` (округлённо, не точное число)
